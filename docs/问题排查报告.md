## 一、问题分类

### 1.1 高优先级问题（P0 - 必须立即修复）

#### 问题1：缺少严格的线框容器控制
**描述**：
- 原始版本有严格的移动端容器限制（375px宽度）
- 现代版本使用了 `max-w-md`（约448px），不够精确
- 导致在不同设备上显示不一致

**影响**：
- 布局可能在某些设备上错位
- 不符合真实移动端APP的视觉效果

**解决方案**：
```css
/* 原始版本 */
max-width: 375px;

/* 现代版本（需要改） */
max-w-md (448px) → max-w-[375px]
```

**涉及文件**：
- `modern/modern.html`
- `modern/交易查询.html`
- `modern/个贷场景-贷前.html`
- `modern/理财场景.html`
- 所有其他页面

---

#### 问题2：缺少移动端视口容器
**描述**：
- 原始版本有完整的手机外壳模拟
- 现代版本只有简单的 `max-w-md`，缺少手机框架
- 在桌面浏览器上查看时不够真实

**影响**：
- 无法直观看到移动端的真实效果
- 容易忽略移动端的限制

**解决方案**：
- 在 `index.html` 的 iframe 中已经有手机外壳
- 但现代页面本身也需要考虑移动端限制

#### 问题4：布局不符合移动端APP规范
**描述**：
- 某些间距、字号、按钮大小不符合移动端标准
- 缺少移动端特有的交互反馈

**具体问题**：

**a. 字号问题**
- 原始版本：标题16-18px，正文13-14px
- 现代版本：标题20-24px，正文15-17px（偏大）

**b. 间距问题**
- 原始版本：紧凑布局，间距8-12px
- 现代版本：宽松布局，间距16-24px（偏大）

**c. 按钮高度**
- 原始版本：按钮高度32-40px
- 现代版本：按钮高度44-48px（偏高）

**解决方案**：
- 统一使用移动端标准尺寸
- 参考 iOS Human Interface Guidelines
